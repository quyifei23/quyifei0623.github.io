Hexo搭建流程

流程详解

1. hexo init

等价于两行shell指令。

  1. git clone –recursive https://github.com/hexojs/hexo-starter.git blog

将仓库中的全部内容复制到当前文件夹下，相当于配置当前文件夹。注意如果使用 hexo init ,指的是配置当前所在的文件夹，并且要使得当前文件夹为空，如果使用的是 hexo init blog 则是创建一个新的文件夹 blog 并配置。配置是指创建一些基础默认的文件和文件夹。

  1. npm install –production

详解：

hexo

这里的 hexo 命令是 nodejs 脚本，原来的 /usr/local/bin/hexo 是一个链接符号，链接到 nodejs 的 modules 目录中。

即 hexo 命令对应的是/usr/local/lib/node_modules/hexo-cli/bin/hexo.js文件。

而 init 对应的是 node_modules/hexo-cli/lib/console/ 下的四个原始 .js 文件
